## 1. 系统概述

「蠢驴电竞陪玩派单管理系统」是一套面向电竞陪玩工作室的综合管理平台，覆盖 **接单、派单、报账、客户管理、员工管理、盈亏统计** 等全业务流程。系统支持多角色（总管理员、管理员、客服、陪玩）协同工作，实现陪玩工作室的数字化运营。

**核心目标：**
- 统一管理陪玩订单的派发、跟踪和结算
- 实时监控陪玩电脑状态与工作产出
- 自动化报账、流水统计与收入分润
- 老板端可视化盈亏分析

---

## 2. 用户角色与权限

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| **总管理员** | 工作室老板（韩磊） | 全部权限，含盈亏统计（需二级密码） |
| **管理员** | 工作室运营管理者 | 客户管理、收入管理、电脑控制、派单管理 |
| **客服** | 派单操作人员 | 派单、对话界面、查看陪玩状态 |
| **陪玩** | 具体执行陪玩的工作人员 | 查看自己的客户、自己的收入、电脑状态、接单 |

**多层权限分层：**
- 老板端：后台显示所有数据
- 陪玩端：只能看到自己的客户和收入
- 客服端：需要老板授权才能登录客户端

## 3. 功能模块

### 3.1 身份与权限管理

#### 3.1.1 角色定义
- 总管理员（老板）
- 工作室管理员
- 客服
- 陪玩

#### 3.1.2 组织架构
```
总管理员（韩磊）
  └── 工作室
       ├── 管理员
       ├── 客服
       └── 陪玩
```

#### 3.1.3 权限要求
- 客服客户端：需要老板授权
- 陪玩客户端：需要老板授权
- 工作室盈亏统计：需要二级密码登录

---

### 3.2 报账系统

#### 3.2.1 订单类型与报账
系统需支持以下四种订单类型的报账：

| 订单类型 | 报账内容 |
|----------|----------|
| **新单** | 日期 + 微信/支付宝付款 + 客户转账截图上传 + 客户信息（微信号/客户编号） |
| **续单** | 日期 + 客户转账截图上传 + 客户信息（微信号/客户编号） |
| **复购** | 日期 + 客户转账截图上传 + 客户信息（微信号/客户编号） |
| **打赏** | 日期 + 客户转账截图上传 + 客户信息（微信号/客户编号） |

#### 3.2.2 流水统计

**日总流水：**
- 显示维度：日期
- 统计项：新单数量/金额/占比、续单数量/金额/占比、复购数量/金额/占比

**月总流水：**
- 显示维度：月份
- 统计项：新单数量/金额/占比、续单数量/金额/占比、复购数量/金额/占比
- 附加项：陪玩当月收入

#### 3.2.3 报账码
- 每个陪玩拥有唯一的「陪玩报账码」，用于报账流程的关联与追溯

#### 3.2.4 收入汇总
- 总收入 + 每个陪玩的收入（按日/月展示）
- 自动排名

---

### 3.3 陪玩电脑管理系统

#### 3.3.1 陪玩信息
- 擅长游戏：支持自定义
- 自行选择当前状态

#### 3.3.2 状态管理
陪玩可在客户端设置自己的状态，顶部实时显示（可选择展开/收起）：

| 状态 | 颜色 | 说明 |
|------|------|------|
| **接单中** | 🟥 红色 | 当前可接单 |
| **空闲中** | 🟩 绿色 | 来单提示 |
| **其他** | 🟨 黄色 | 自定义状态 |

每种状态需关联 **时长计时**。

#### 3.3.3 开机自主选择模式

**娱乐模式：**
- 陪玩自行使用的模式
- 限流功能：后台可设置限速（xx kb/s）

**接单模式：**
- 正式开始接单
- 后台开始统计接单时间
- 统计数据同步显示到：陪玩端 + 客服端 + 管理端
- 网络恢复正常流量

#### 3.3.4 工作数据统计
后台需统计并显示：
- 每个陪玩（ABC…）的娱乐时间
- 每个陪玩的接单时间
- 每台电脑运行总时长
- 最终与报账金额 + 小时数做对比校验
- 统计结果同时在「后台」和「陪玩端」展示

#### 3.3.5 弹窗系统

**空闲 + 其他状态下的弹窗：**
- 客服指派的订单→大窗弹出
- 陪玩点击「确定」后弹窗消失

**接单模式下的弹窗：**
- 客服指派的订单→右下角小窗弹出
- 陪玩点击后消失

### 3.4 派单管理系统（客服端）

#### 3.4.1 陪玩状态面板
- 最顶部实时显示所有陪玩状态
- 支持展开/收起

#### 3.4.2 客服对话界面
- 支持多工作室管理：
  - 工作室A
  - 工作室B
- 支持按工作室分别管理客服和陪玩

#### 3.4.3 派单操作

**方式一：直接派到客户池**
- 所有在线陪玩可见
- 包含订单信息：游戏名称、时长、模式等（支持自定义字段）
- 陪玩抢单后订单自动消失
- 订单归属到「对应工作室→对应陪玩」的客户系统下

**方式二：派给指定陪玩**
- 直接指派给某个陪玩
- 陪玩确认后订单自动消失
- 归属到该陪玩的客户系统下

#### 3.4.4 派单记录
- **线上派单记录**：线上派单的收入明细、入账记录（转账图片），管理员确认
- **线下派单记录**：同上，记录线下派单的收入

---

### 3.5 客户管理系统（老板端）

#### 3.5.1 客户信息字段

| 字段 | 说明 |
|------|------|
| **客户编号** | 系统唯一标识 |
| **微信号** | 客户的微信号 |
| **所属陪玩 + 消费金额** | 关联陪玩及累计消费 |
| **所属平台** | 自定义选择：小红书 / 抖音 / 快手 / 视频号 / B站 |
| **所属账号** | 平台上的具体账号 |
| **咨询日期** | 首次咨询的日期 |
| **账号是否封禁** | 是/否标记 |
| **加到微信日期** | 客户添加微信的日期 |
| **是否被客户删除** | 是/否标记 |
| **备注** | 自由文本 |

#### 3.5.2 接单记录
- 每个客户的完整接单历史
- 支持自定义字段扩展

#### 3.5.3 权限隔离
- 老板端：后台显示所有客户的完整信息
- 陪玩端：仅显示自己名下客户的信息

---

### 3.6 员工管理系统

#### 3.6.1 员工管理
- 员工示例：张三、李四…（支持多名陪玩）
- 每个陪玩可自己设置登录密码
- 密码在后台对管理员可见

#### 3.6.2 陪玩首页
每个陪玩的个人首页显示：
- 本月收入
- 自己昨日收入
- 工作室所有人收入排名

#### 3.6.3 陪玩端功能
- 收入管理 → 流水 / 押金 / 支取 / 支取记录
- 客户管理（仅自己名下客户）
- 收入记录 + 截图查看

---

### 3.7 工作室盈亏统计（老板端）

> ⚠️ **安全要求**：需要输入二级密码登录

#### 3.7.1 收入统计

| 维度 | 内容 |
|------|------|
| **线上收入** | 收入记录 + 截图 |
| **线下收入** | 收入记录 + 截图 |
| **本日收入** | 当日汇总 |
| **本月收入** | 当月汇总 |

#### 3.7.2 支出统计
- 本月总支出

#### 3.7.3 盈亏计算
- 本月盈亏 = 本月收入 - 本月支出

---

### 3.8 管理员远程控制

管理员和客服可对每一台陪玩电脑进行远程控制：
- **开关机**：远程控制陪玩电脑的开关机
- **限速**：设置带宽限速（xx kb/s）
- **隐藏运行**：控制程序在后台静默运行
- **低资源占用**：控制程序尽量少占用系统资源

## 4. 业务规则

### 4.1 收入分润规则
- 陪玩当月收入 **不超过一万元**：分成比例为 **60%**
- 陪玩当月收入 **超过一万元**：分成比例为 **70%**

### 4.2 订单归属
- 客服派单到客户池后，陪玩抢单成功即归属
- 客服指定派单后，陪玩确认即归属
- 归属后订单进入该陪玩的客户系统

### 4.3 状态流转
```
空闲中 → （客服派单）→ 接单中 → （订单完成）→ 空闲中
接单中 → （自行切换）→ 其他
空闲中 → （自行切换）→ 其他
其他 → （自行切换）→ 空闲中 / 接单中
```

### 4.4 报账审核
- 陪玩提交报账 → 上传转账截图 → 管理员确认 → 入账

## 5. 非功能性需求

### 5.1 安全性
- 老板盈亏统计需要二级密码登录
- 客户端登录需老板授权
- 密码后台可见（管理员视角）

### 5.2 性能
- 管理员远程控制程序需 **隐藏运行，少占用系统资源**
- 限流功能需精确控制带宽（xx kb/s）

### 5.3 多端同步
- 陪玩状态需实时同步到：陪玩端、客服端、管理端
- 订单弹窗需即时推送到陪玩端

### 5.4 数据可视化
- 流水统计（日月维度）
- 陪玩收入排名
- 盈亏报表

### 5.5 多工作室支持
- 系统需支持多个工作室（A、B…）独立管理
- 客服按工作室分组
