## 1. 系统概述

「蠢驴电竞陪玩派单管理系统」是一套面向电竞陪玩工作室的综合管理平台，覆盖接单、派单、报账、客户管理、员工管理、盈亏统计等全业务流程。

### 1.1 用户角色

| 角色 | 访问方式 | 权限范围 |
|------|---------|---------|
| 总管理员（老板） | 浏览器 Web | 全部权限，含盈亏统计（需二级密码） |
| 管理员 | 浏览器 Web | 客户管理、收入管理、电脑控制、派单管理 |
| 客服 | 浏览器 Web | 派单、对话界面、查看陪玩状态 |
| 陪玩 | Go Agent 桌面客户端 | 查看自己的客户/收入、电脑状态、接单 |

### 1.2 功能模块

| 模块 | 说明 |
|------|------|
| 身份与权限管理 | 4 角色 RBAC + 老板授权 + 二级密码 |
| 报账系统 | 订单报账、流水统计（日/月）、收入排名 |
| 陪玩电脑管理 | 状态切换、接单/娱乐计时、弹窗系统 |
| 派单管理 | 派单池抢单 + 指定派单、客服工作台 |
| 客户管理 | 客户信息、接单历史、归属陪玩、按平台追踪 |
| 员工管理 | 员工列表、收入排名、密码管理 |
| 盈亏统计 | 收入/支出/盈亏计算（需二级密码） |
| 远程控制 | 开关机、带宽限速、Agent 静默运行 |

### 1.3 跨工作室场景

系统支持多工作室（A、B…）独立管理，同时支持跨工作室派单——一个工作室的客服可将订单分发到其他工作室的陪玩池。

## 7. Go Agent 设计

### 7.1 核心模块

| 模块 | 职责 |
|------|------|
| 计时引擎 | 追踪接单/娱乐模式切换，累计秒数随心跳上报 |
| 心跳上报 | 每 30 秒发一次，携带状态、模式、计时数据 |
| 指令执行 | 接收并执行远程指令（关机、限速、重启） |
| 弹窗通知 | 新订单推送时弹出窗口（空闲大窗 / 接单小窗） |
| 网络控制 | Windows QoS API 实现带宽限速 |
| 本地 Web UI | localhost HTTP 服务，WebView2 渲染 |

### 7.2 Agent 生命周期

1. **安装**：陪玩运行 installer → 注册 Windows Service
2. **首次启动**：弹出登录窗口 → 输入账号密码 → 获取 JWT → 加密存本地
3. **正常运行**：Service 自动拉起 → 读 Token → 连接 WebSocket → 开始心跳
4. **模式切换**：陪玩点 Web UI 切换 → 本地计时引擎记录 → 下个心跳上报
5. **限速触达**：管理员下发指令 → Agent 执行 QoS Policy → 回复 ack
6. **更新**：心跳回带版本号 → 后端判断 → 下发新版本 URL → 下载 → 重启

### 7.3 安全设计

- Token 加密存储本地文件
- WebSocket 连接使用 JWT 认证
- 远程指令需验证来源 server 签名
- Windows Service 运行，崩溃自动重启
- 编译后单文件 < 15MB，内存占用 < 50MB

## 8. 实时通信

### 8.1 WebSocket 房间分组

| 房间 | 成员 | 用途 |
|------|------|------|
| `studio:{id}` | 该工作室的管理员、客服 | 状态广播、订单池更新 |
| `companion:{id}` | 单个陪玩 + 管理员 | 订单推送、指令下发 |
| `pc:{id}` | 单个 Agent + 管理员 | 远程指令 + ack |

### 8.2 事件列表

| 事件 | 方向 | 说明 |
|------|------|------|
| `companion:status` | Agent → Server | 状态变更上报 |
| `companion:heartbeat` | Agent → Server | 30s 心跳 + 计时数据 |
| `order:new` | Server → Agent | 新订单推送（触发弹窗） |
| `order:grabbed` | Server → CS | 订单被抢走 → 池中移除 |
| `order:confirmed` | Server → CS | 指定派单被确认 |
| `pc:command` | Server → Agent | 远程指令 |
| `pc:command_ack` | Agent → Server | 指令确认 |
| `status:broadcast` | Server → Studio | 陪玩状态批量广播 |

## 9. 安全设计

### 9.1 认证

- **JWT 双 Token**：accessToken 15 分钟 + refreshToken 7 天
- **Cookie httpOnly** 存 refreshToken，accessToken 存内存
- **二级密码**：独立短期 secondToken（5 分钟），仅盈亏统计接口校验

### 9.2 授权

- **RBAC**：OWNER > ADMIN > CS > COMPANION 四级权限
- **客户端授权**：客服/陪玩需老板在后台手动授权后才能登录
- **数据隔离**：陪玩只能看见自己的客户和收入
- **工作室内隔离**：按 studioId 过滤所有查询
