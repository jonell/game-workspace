核心模块详解

⏱️ 计时引擎

追踪陪玩在不同模式下的累计时间，随心跳上报：

type TimeTracker struct {
    CurrentMode    string        // WORK / ENTERTAINMENT
    ModeStartedAt  time.Time     // 当前模式开始时间
    WorkSeconds    int64         // 累计接单秒数
    EntertainSeconds int64       // 累计娱乐秒数
    TotalRuntime   int64         // 总运行秒数
}

// 心跳携带数据
type Heartbeat struct {
    CompanionID string
    Status      string  // ONLINE / BUSY / IDLE
    Mode        string
    WorkSec     int64
    EntertainSec int64
    Timestamp   int64
}

每次心跳上报累计秒数，后端持久化到 CompanionTimeLog。

🌐 网络限速控制

两层策略：

策略实现适用
PC 本机 Windows QoS API / netsh 单机精确限速
路由器 SSH 登路由器执行 tc/iptables 整网段控制

优先用本机 QoS，路由器 SSH 作为备用方案。

🖥️ 远程控制指令
指令Windows 实现
关机 shutdown /s /t 0 或 InitiateSystemShutdownEx
重启 shutdown /r /t 0
限速 QoS Policy 设置出站带宽上限
取消限速 移除 QoS Policy
隐藏运行 注册为 Windows Service，无窗口
🛡️ 安全与自保
  • Token 认证：首次启动弹出登录窗口 → 获取 JWT → 加密存本地
  • 自启动：注册 Windows Service，开机自动运行
  • 崩溃恢复：守护进程监控，异常退出自动重启
  • 版本更新：心跳回带版本号，后端推送新版本下载 URL
  • 低资源：Go 编译单文件 <15MB，内存占用 <50MB
  • 静默：无托盘图标时可完全隐藏，仅 WebView 供陪玩交互

Agent 生命周期

  安装:  陪玩运行 installer → 注册 Windows Service → 弹出登录 → 输入账号密码 → 获取 Token
  启动:  Service 自动拉起 → 读取本地 Token → 连接 WebSocket → 上报心跳
  运行:  每 30s 心跳 → 监听 WS 指令 → 本地 HTTP 提供 Web UI → 计时引擎持续记录
  模式切换: 陪玩点击 Web UI 切换 娱乐/接单 → 本地计时引擎记录切换点 → 下个心跳上报
  限速触达: 管理员下发限速指令 → Agent 执行 QoS Policy → 回复 ack → 心跳持续报告限速状态
  更新:  心跳回带版本号 → 后端判断需更新 → 下发新版本 URL → Agent 下载 → 重启自身
