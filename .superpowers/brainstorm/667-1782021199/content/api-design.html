📡 WebSocket 事件

实时状态同步，房间按 studioId 分组

事件名 方向 房间 说明
companion:status Agent → Server studio:{id} 陪玩状态变更 (接单中/空闲/其他)
companion:heartbeat Agent → Server — 30s 心跳 + mode + 计时数据
order:new Server → Agent companion:{id} 新订单推送（弹窗）
order:grabbed Server → CS studio:{id} 订单被抢走 → 池中移除
order:confirmed Server → CS studio:{id} 指定派单被确认
pc:command Server → Agent pc:{id} 远程指令：关机/限速/重启
pc:command_ack Agent → Server pc:{id} 指令执行确认
🔄 鉴权流 (JWT + WebSocket)
  登录:   Client → POST /auth/login → JWT(access 15min + refresh 7d)
  REST:   Authorization: Bearer <access_token>
  WS:     io("ws://host", { auth: { token: "<access_token>" } })
          → 服务端 connect 时验证 JWT → join("studio:${user.studioId}")
          → 陪玩额外 join("companion:${user.companionId}")

  二级密码: Client → POST /auth/verify-2nd → 返回短期 secondToken(5min)
           → 盈亏接口需 Header: X-Second-Token
